# Synthetic Tickets Dataset

200 IT support tickets generated with a fixed seed (123) for reproducibility.

**Categories:** access, hardware, software, network, account, security — sampled uniformly.
**Statuses:** resolved (60%), open (25%), escalated (15%).
**Requesters:** 20 fictitious usernames with unequal weights (a few repeat offenders).
**Teams:** assigned per category with weighted probabilities (e.g. security tickets → Security 90%, IT Ops 10%).
**Dates:** business-hours timestamps between 2025-01-06 and 2026-06-02.

Text fields (subject, description, resolution) are generated by picking a category-specific template and filling named slots (`{system}`, `{app}`, etc.) from fixed word lists. ~8% of tickets include a cross-reference to a prior resolved ticket.

Run `poe generate-data` or `python -m ticket_generation` from the `scripts/` directory to regenerate.
